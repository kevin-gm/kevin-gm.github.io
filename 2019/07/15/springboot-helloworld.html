SpringBoot之HelloWorld

2019-07-14      标签 springboot

SpringBoot的简单介绍

学习使用一项新技术，笔者觉得大致可以分为三个阶段

  • 首先了解这项技术是什么，能解决什么问题
  • 其次是，通过各种资料，能搭建或使用这项技术
  • 最后才是如何才能用好，也就是了解原理性的知识。

所以，本文的开头，先来说说笔者眼里的SpringBoot。

在此之前，我们先回忆一下，以前我们搭建一个SpringMVC项目时，需要做些什么工作。在我的记忆里，需要这么几个步骤

  • 引入依赖。因为是SpringMVC项目，所以有几个基本依赖是必须的：spring-beans,spring-core,spring-context,spring-web,spring-webmvc，以及其他必须的依赖。
  • spring配置文件的定义和配置。默认是classpath下的applicationContext.xml，可以自己重命名并且放到其他路径下，这个文件主要是配置需要加载的Bean，可以是 bean 标签配置，也可以配置自动挡扫描 component-scan。
  • springmvc配置文件的定义和配置。该文件主要是配置与mvc相关的，比如controller的包，viewResolver，messageConverter等等。
  • 还需要配置web.xml文件，这里需要指明 spring配置文件，dispatchservlet以及对应springmvc配置文件，spring与servlet容器桥接的contextListener等等。
  • 最后，历经千辛万苦，定义一个controller类，定义一个hello的方法，返回”hello world”字符串，实现我们的HelloWorld

以上，就是一个简单SpringMVC应用的传统搭建。很明显，只有最后一个步骤，定义HelloWorld方法才是我们真正想要实现的，传统的方式，我们做了大量的辅助工作来支撑环境的运行，看完后面的入门demo，感受会更加深刻。

SpringBoot的出现，就是为了解决上述问题，将开发人员从各种依赖配置中解放出来，从而能专注于业务需求。SpringBoot可以轻松的创建基于spring的，可以独立运行的生产级项目，其预先定义了很多组件的依赖，并实现其自动装配，所以一般只需要一些小改动就可以实际运行。

总的来说，SpringBoot并不算是一个新的框架，更明确的说，它更像是Spring的高级运用，是基于Spring的一套脚手架。所以，要彻底的了解SpringBoot，需要预先了解Spring才行。

SpringBoot之HelloWorld

工具及版本:

  • IDEA：2018.2
  • JDK： 1.8.0_171
  • MAVEN：3.5.3
  • spring-boot：2.1.6.RELEASE

Notice：请注意，SpringBoot 2.x 之后对应的Spring版本为 5.x，因其大量使用了JDK8的新特性，诸如lambda表达式，stream api等，因此JDK版本必须大于等于1.8。

使用spring initializr创建springboot应用

使用spring initializr有两种方式，第一种是通过官方提供的控制台，如下图：

这里可以按需选择项目的信息，dependencies可以选择项目的依赖，本demo只需要选择web。

选择好之后，点击”Generate the project”，控制台将生成并下载项目到本地，解压缩后导入项目到IDEA即可。

第二种方式是通过IDEA创建。在IDEA的菜单栏，选择 new -> Project，在弹出的界面中，选择左边的 Spring Initializr，如图所示：

稍微要注意的是JDK的版本至少需要1.8，可以看到，这里的服务地址实际还是使用的官方的控制台，只不过在IDEA创建，不需要额外的下载导入步骤。点击 next，选择所需要的依赖，同样，我们只选择web，点击 finish，静等项目创建完毕。

通过创建的项目，其 pom.xml文件内容如下：

<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
    xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">
    <modelVersion>4.0.0</modelVersion>
    <parent>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-parent</artifactId>
        <version>2.1.6.RELEASE</version>
        <relativePath/> <!-- lookup parent from repository -->
    </parent>
    <groupId>site.wellman.study</groupId>
    <artifactId>springboot</artifactId>
    <version>0.0.1-SNAPSHOT</version>
    <name>springboot</name>
    <description>Demo project for Spring Boot</description>

    <properties>
        <java.version>1.8</java.version>
    </properties>

    <dependencies>
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-web</artifactId>
        </dependency>

        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-test</artifactId>
            <scope>test</scope>
        </dependency>
    </dependencies>

    <build>
        <plugins>
            <plugin>
                <groupId>org.springframework.boot</groupId>
                <artifactId>spring-boot-maven-plugin</artifactId>
            </plugin>
        </plugins>
    </build>

</project>

注意，前面的pom文件，是指定了父pom为spring-boot-starter-parent，所以后面的依赖可以不指定版本。

通过普通的maven项目创建springboot应用

直接通过Spring Initializr很方便，能直接按需选择所需的依赖，但每次创建都会是一个新的 Project，在实际的开发过程中，通常都是一个父Project，下面包含多个子module，下面我们通过这种方式来新建一个springboot应用。

首先，通过IDEA新建一个普通的maven项目，其pom.xml文件配置如下：

<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">
    <parent>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-parent</artifactId>
        <version>2.1.6.RELEASE</version>
    </parent>
    <modelVersion>4.0.0</modelVersion>

    <groupId>site.wellman.study</groupId>
    <artifactId>spring-boot</artifactId>
    <packaging>pom</packaging>
    <version>1.0-SNAPSHOT</version>
    <modules>
        <module>HelloWorld</module>
    </modules>

    <build>
        <plugins>
            <plugin>
                <groupId>org.springframework.boot</groupId>
                <artifactId>spring-boot-maven-plugin</artifactId>
            </plugin>
        </plugins>
    </build>
</project>

当然，一般我们会将父project项目设定为顶层项目，不希望在父project项目中还引入parent标签，此时我们可以调整pom.xml为：

<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">

    <modelVersion>4.0.0</modelVersion>

    <groupId>site.wellman.study</groupId>
    <artifactId>spring-boot</artifactId>
    <packaging>pom</packaging>
    <version>1.0-SNAPSHOT</version>
    <modules>
        <module>HelloWorld</module>
    </modules>

    <dependencyManagement>
        <dependencies>
            <dependency>
                <groupId>org.springframework.boot</groupId>
                <artifactId>spring-boot-starter-parent</artifactId>
                <version>2.1.6.RELEASE</version>
                <type>pom</type>
                <scope>import</scope>
            </dependency>
        </dependencies>
    </dependencyManagement>
</project>

通过 dependencyManagement来引入spring-boot-starter-parent

新建好父project后，然后在父project下新建子模块，也是一个普通的maven项目，配置其pom.xml

<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0 http://maven.apache.org/xsd/maven-4.0.0.xsd">
    <parent>
        <artifactId>spring-boot</artifactId>
        <groupId>site.wellman.study</groupId>
        <version>1.0-SNAPSHOT</version>
    </parent>
    <modelVersion>4.0.0</modelVersion>

    <artifactId>HelloWorld</artifactId>

    <dependencies>
        <dependency>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-starter-web</artifactId>
        </dependency>
    </dependencies>

</project>

我们的目标是一个HelloWorld级的SpringMVC应用，所以只需要引入spring-boot-starter-web

编写启动类代码以及controller，代码如下：

package site.wellman.study.springboot;

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RestController;

@SpringBootApplication
@RestController
public class HelloWorldApplication {

    @GetMapping("/hello")
    public String hello() {
        return "hello world";
    }

    public static void main(String[] args) {
        SpringApplication.run(HelloWorldApplication.class, args);
    }
}

最后，启动main方法，应用默认已 8080端口启动，访问controller http://localhost:8080/hello 显示返回的字符串”hello world”

总结来看，当使用springboot构建SpringMVC项目时，我们仅需要引入spring-boot-starter-*依赖，然后编写启动类即可。
