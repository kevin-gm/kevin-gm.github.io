使用IDEA管理数据库

2019-08-18      标签 IDEA Database

前置条件

  • IDEA 2019.2
  • 本机已安装Mysql数据库
  • Mysql 5.7

使用IDEA连接数据库

不得不说，IDEA真是开发的利器。

对于个人学习而言，虽然现在市面上有很多数据库管理工具，比如Navicat，HeidiSQL，SQLyog，Workbench等等，或者直接使用命令行。诸如这些工具，有的界面不好看，有的用得不习惯，我之前用Navicat比较多，也已经习惯了它的界面操作，但后来收费后，每次安装都需要找破解版，十分麻烦。而使用IDEA，直接在开发工具就可以进行数据库操作，方便至极。

配置数据库连接

首先找到并进行IDEA连接数据库的配置。如下图所示，一般IDEA默认在右边的侧栏就已经显示了这个操作菜单，如果没有，那就通过顶部导航栏的 View -> Tool Windows -> Database 打开配置选项

接着，配置数据库连接。如下图所示，点击 + 号(新建按钮)，在 Data Source 选项下，选择 Mysql

选择了目标数据源后，就会弹出对应的配置界面。

第一次配置的时候，界面底部可能会提示类似于 缺少数据库驱动，需要进行下载。这里要注意的是，驱动必须要与安装的数据库版本匹配，2019.2版本的IDEA下载的时候，默认下载的是Mysql8的驱动，这个时候如果安装的Mysql版本没有这么高，则会连接失败。

这里，本人安装的Mysql是5.7的，下载的数据库驱动为5.1.47，能连接成功

填写好配置信息后，可以进行连接测试，如果通过，点击确认。

使用IDEA管理数据库

配置好后，先看看怎么直接执行SQL命令。

点击 console 图标，在SQL命令窗口栏输入简单的SQL命令 show databases; 查看效果，如下图所示。

可以看到，在 Output 一栏显示了当前数据库服务器已经创建好的数据库。

当然也有图形化界面，在 Database 管理区域，如图所示操作，打开需要使用的数据库 practice

选择某一张数据表后，在左侧，会显示数据表当前的数据，可以双击修改数据，然后点击 submit 图标进行提交。

至于其他操作，可以根据提示慢慢摸索。
